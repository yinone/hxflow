# 模块质量评级

> 每双周更新一次，由熵扫描报告驱动。
> 评级标准：A = 无已知问题，B = 有轻微技术债，C = 需要优先清理

| 模块 | 层级 | 评级 | 主要问题 | 最后审查 |
|------|------|------|----------|----------|
| （示例）authService | Service | A | 无 | — |

## 评级标准

**A 级** — 符合所有黄金原则，测试覆盖完整，文档同步
**B 级** — 有 1-3 个 🟡 建议修复问题，不影响功能
**C 级** — 有 🔴 必须修复问题，或文档严重过期，或测试缺失

## C 级处理流程

1. 在下一个 Sprint 创建对应的清理 TASK
2. 运行 `npm run hx:entropy` 获取具体问题列表
3. 用 Agent 批量修复后更新本文档评级
