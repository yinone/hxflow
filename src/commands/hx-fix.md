# Phase 05 · 修复错误

## 目标

- 根据明确的错误上下文直接修复问题，并补上最小验证。

## 何时使用

- 适用场景：已经拿到 review 问题、错误日志或失败文件，需要快速修复并验证。
- 不适用场景：还没有明确错误上下文时，先跑 `hx-review` 或 `hx-qa` 收集问题。

## 输入

- 命令参数：`$ARGUMENTS`
- 必选参数：无
- 可选参数：`--log <text>`、`--file <path>`
- 默认值：未传参数时，自动运行 `test` gate 并截取失败输出
- 依赖输入：错误日志、失败文件路径或自动执行的 `test` gate 输出、`rules/golden-rules.md`

## 执行步骤

1. 获取错误上下文，优先级依次为 `--file <path>`、`--log <text>`、自动运行 `test` gate 的失败输出。
2. 读取 `rules/golden-rules.md`。
3. 根据错误上下文定位相关源文件并直接修复。
4. 运行 `hx-qa` 验证修复结果，并补一个能复现该问题的回归测试。

## 成功结果

- 输出修复摘要、受影响文件和回归验证结果。

## 失败边界

- 错误上下文不足，无法稳定定位修复点。
- 修复后 `hx-qa` 仍未通过。

## 下一步

- 修复成功后继续运行 `hx-review` 或后续交付流程。

## 约束

- 不修改已有函数签名（参数与返回类型）
- 不修改现有测试的期望值（测试是行为契约）
- 修复后补充一个能复现此 Bug 的回归测试
