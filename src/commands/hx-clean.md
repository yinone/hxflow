# Phase 07 · 工程清理扫描

## 目标

- 在交付前补做一轮工程卫生扫描，发现代码异味和文档一致性问题。

## 何时使用

- 适用场景：`hx-qa` 通过后，想在提交前再做一轮静态清理检查。
- 不适用场景：只是验证 gate 是否通过时，优先用 `hx-qa`；只是看 review 问题时，优先用 `hx-review`。

## 输入

- 命令参数：`$ARGUMENTS`
- 必选参数：无
- 可选参数：无
- 默认值：无
- 依赖输入：当前项目 `.hx/config.yaml`、源码目录、需求与进度文档

## 执行步骤

1. 读取 `.hx/config.yaml` 中的 `paths`，确定扫描范围。
2. 扫描主源码目录，排除测试文件。
3. 检查典型工程问题，如 `console` 调用、裸 `throw new Error(...)`、`: any`、可疑断言、空 `catch`、`TODO / FIXME / HACK / XXX`。
4. 检查 `progressFile` 与需求文档、代码状态之间是否明显脱节。
5. 输出分类报告，区分代码问题和文档问题。

## 成功结果

- 输出一份可执行的清理报告，标记问题类型、范围和后续处理建议。

## 失败边界

- 无法定位项目配置，或源码目录不可读。
- 扫描前提不满足，无法形成有效报告。

## 下一步

- 存在问题时，运行 `hx-fix` 或人工修复；全部通过时进入提交流程。

## 约束

- 只基于当前项目内的配置、代码和文档输出结果
- 不修改任何文件，只做扫描和报告
