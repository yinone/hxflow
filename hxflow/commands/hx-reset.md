# 重置需求产物

## 执行步骤

1. 执行 `bun scripts/tools/reset.ts <feature> [plan|doc|code]`，删除或回退目标 feature 的对应产物。
2. 根据脚本返回的结果确认重置范围；未指定参数时按同时重置 `code`、`plan` 和 `doc` 处理。

## 约束

- `plan` 只重置计划相关产物，不删除需求文档
- `doc` 同时重置需求文档及其下游计划产物
- `code` 回退 requirement 首次创建之后的代码改动，并保留需求文档和计划文档
- 无参数时执行全量重置：先回退代码，再删除需求和计划产物
- 不保留会影响后续状态判断的归档产物
